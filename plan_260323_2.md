## 문제 1: Residue Info 섹션이 두 번 표시됨

### 현상
스크린샷에서 패널 하단에 "Residue Info" 블록이 두 번 보인다:
1. 패널 내부의 정상 위치 (bottom border `*===*` 바로 위)
2. 패널 아래에 한 번 더 (bottom border 밖)

### 근본 원인
`Screen.cpp:919-926`의 `update_hover_info()` 마우스 이벤트 핸들러에서, hover가 발생할 때마다 `draw_hover_section()`을 **별도로** 호출하여 부분 갱신한다.

문제는 `hover_row` 계산에 있다:
```cpp
int hover_row = last_panel_start_row + last_panel_h - 1 - rsh;
```
이 계산이 실제 `draw_panel()`에서 Residue Info를 그린 행과 **다를 수 있다**. `get_height_for_width()`와 `draw_panel()`에서 entry 수가 많을 때 (9개 단백질), entry별 blank line 등으로 `draw_panel()` 내의 실제 Residue Info 시작 행과 `last_panel_h`에서 역산한 행이 불일치하면, **다른 행에 Residue Info가 한 번 더 그려진다**.

구체적으로:
- `draw_panel()`이 그린 Residue Info: `draw_panel()` 내부에서 `r` 값이 진행되면서 결정
- `update_hover_info()`가 그리는 Residue Info: `last_panel_start_row + last_panel_h - 1 - rsh`로 역산

`get_height_for_width()`의 높이 계산과 `draw_panel()`의 실제 렌더링이 정확히 일치하면 같은 위치지만, **각 entry 뒤에 blank line(`++r; clear_line(r); ++r;`)이 `get_height_for_width()`에서 `lines += 2`로 계산된다**. chain 줄 수 계산 (`count_wrapped_lines_for_chaininfo`)과 실제 `draw_panel()`의 줄 바꿈 로직이 일치하지 않으면 offset이 어긋난다.

또한, `draw_panel()`에서 `in_rows(r)` 체크로 조기 종료되면 Residue Info가 그려지지 않는 반면, `update_hover_info()`의 `draw_hover_section()`은 독립적으로 그리므로 **패널 바깥에 Residue Info가 나타날 수 있다**.

### 수정 방안

**`draw_panel()` 내에서 Residue Info 섹션의 실제 시작 행을 기록**하고, `update_hover_info()`에서 그 값을 사용한다.

**수정 파일: `Panel.hpp`, `Panel.cpp`, `Screen.cpp`**

1. `Panel` 클래스에 `mutable int last_hover_row = -1;` 멤버 추가
2. `draw_panel()` 내에서 Residue Info를 그리기 직전에 `last_hover_row = r;` 저장
3. `draw_hover_section()` 호출 전에 `panel->get_last_hover_row()`로 정확한 행 사용
4. `last_hover_row == -1`이면 (패널이 그려지지 않았거나 Residue Info까지 도달 못한 경우) hover 갱신 스킵

## 문제 2: 3D 시각화 영역이 너무 좁음

### 현상
9개 단백질을 로드하면 패널이 화면의 절반 이상을 차지하여 3D 렌더링 영역이 극히 좁아진다.

### 근본 원인
`draw_screen()` (`Screen.cpp:744-795`)에서:
```cpp
int panel_h = panel->get_height_for_width(panel_cols);
offset = panel_h;  // 패널 높이만큼 3D 영역이 위로 밀림
```

9개 단백질 entry = 파일명(1줄) + chain정보(1~2줄) + blank(2줄) = entry당 4~5줄.
9 × 4~5 = 36~45줄 + header(3줄) + Residue Info(5~6줄) + bottom border(1줄) = **총 45~55줄**.

터미널이 예를 들어 60줄이면 패널이 50줄을 차지하고 3D 영역은 10줄밖에 안 남는다.

### 수정 방안

**패널 높이에 상한을 두고, 단백질 목록이 길면 스크롤 가능하게 하거나 요약 표시**한다.

#### 방안 A: 패널 최대 높이 제한 + entry 목록 축소 (추천)

패널 높이가 터미널 높이의 일정 비율(예: 40%)을 초과하면:
1. 각 entry의 blank line(2줄)을 1줄로 줄인다
2. 그래도 초과하면 entry당 chain 정보를 한 줄로 압축 (chain 수만 표시)
3. 그래도 초과하면 파일명만 표시 (chain 정보 생략)

**수정 파일: `Panel.hpp`, `Panel.cpp`, `Screen.cpp`**

1. `Screen::draw_screen()`에서 `panel_h`를 `rows * 0.4` 이하로 제한
2. `Panel`에 compact 모드 플래그를 도입하여, 제한된 높이 안에서 entry를 압축 표시
3. `get_height_for_width()`에도 compact 모드 반영

구체적인 compact 레벨:
- **Level 0** (기본): 현재와 동일 (blank line 2줄, chain 전체 표시)
- **Level 1**: entry간 blank line 1줄로 축소
- **Level 2**: entry간 blank line 제거, chain 정보를 "N chains" 한 줄로 요약
- **Level 3**: 파일명만 1줄로 표시

`draw_panel()`과 `get_height_for_width()` 모두에 compact_level을 받아 일관된 높이를 계산/렌더링한다.

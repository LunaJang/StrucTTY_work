# StrucTTY 화면 깜빡임(Flicker) 제거 계획 (2026-03-23-3)

---

## 문제 현상

매 프레임마다 화면 전체가 깜빡인다. 특히 단백질이 여러 개이거나 구조가 복잡할 때 (렌더링 시간이 길수록) 더 심하게 느껴진다.

---

## 근본 원인 분석

현재 `draw_screen()` (`Screen.cpp:744-810`)의 렌더링 순서:

```
1. clear_screen()          ← logicalPixels 버퍼 초기화 (메모리 연산, 화면 무관)
2. project()               ← 3D→2D 투영 + z-buffer (CPU 계산, 화면 무관)
3. clear()                 ← ★ ncurses 화면 전체를 비움 (터미널에 빈 화면이 보임)
4. print_screen(offset)    ← Braille 문자 출력 (mvaddstr 호출)
5. clrtoeol() 루프          ← 패널 영역 정리
6. draw_panel()            ← 패널 그리기
7. refresh()               ← ★ ncurses 내부 버퍼를 터미널에 실제 출력
```

### 핵심 문제: `clear()` + `refresh()` 사이의 시간 격차

ncurses의 `clear()`는 내부 버퍼를 완전히 비운 뒤 **다음 `refresh()` 때 화면 전체를 다시 그리도록** dirty 플래그를 설정한다. 이것 자체는 즉시 터미널에 반영되지 않지만, **`clear()`가 설정하는 "전체 다시 그리기" 플래그 때문에 `refresh()` 때 모든 셀을 다시 출력**하게 된다.

문제의 진짜 원인은 두 가지:

1. **`clear()` 사용**: `clear()`는 ncurses에게 "전체 화면이 변경되었다"고 알려주므로, `refresh()` 때 **변경되지 않은 셀까지 전부 다시 출력**한다. 이로 인해 터미널에 전송되는 데이터량이 많아지고, 화면 갱신이 느려져 깜빡임이 생긴다.

2. **이전 프레임과의 diff 무효화**: ncurses는 원래 이전 프레임과 현재 프레임의 차이만 터미널에 전송하는 최적화를 내장하고 있다 (`refresh()`가 자동 수행). 하지만 `clear()`를 호출하면 이 최적화가 무효화되어 매번 전체 화면을 다시 전송한다.

### 추가 요인: `clrtoeol()` 루프

```cpp
for (int r = start_row; r < rows; ++r) {
    move(r, 0);
    clrtoeol();
}
```

패널 영역을 비운 뒤 `draw_panel()`로 다시 채우는데, `clear()` 이후에 이미 비어있으므로 이중 작업이다.

## 해결 방안

### 핵심 원칙: `clear()` 제거, ncurses의 diff 최적화 활용

ncurses는 `refresh()` 호출 시 이전 화면과 현재 화면의 차이만 터미널에 전송한다. `clear()`를 쓰지 않으면 이 최적화가 작동하여:
- 변경된 셀만 터미널에 출력됨
- 데이터 전송량 대폭 감소
- 깜빡임 완전 제거

### 구체적 수정

#### 변경 1: `clear()` → `erase()` 교체

```cpp
// Before:
clear();
print_screen(offset);

// After:
erase();
print_screen(offset);
```

**`erase()` vs `clear()` 차이점**:
- `erase()`: 내부 버퍼를 공백으로 채우지만, "전체 다시 그리기" 플래그를 설정하지 않음. `refresh()` 때 ncurses가 이전 화면과 비교하여 **실제 변경된 셀만** 터미널에 출력함.
- `clear()`: `erase()`와 동일하게 버퍼를 비우고 **추가로** `clearok(stdscr, TRUE)`를 호출하여 `refresh()` 때 **모든 셀**을 터미널에 재출력하도록 강제함.

이 한 줄 교체만으로 ncurses의 diff 최적화가 활성화되어 깜빡임이 크게 줄어든다.

#### 변경 2: 패널 영역 `clrtoeol()` 루프 제거

```cpp
// Before:
for (int r = start_row; r < rows; ++r) {
    move(r, 0);
    clrtoeol();
}
if (!no_panel){
    panel->draw_panel(...);
}

// After:
// erase()가 이미 전체 버퍼를 공백으로 채웠으므로,
// 패널 영역을 별도로 지울 필요 없음.
if (!no_panel){
    panel->draw_panel(...);
}
```

`erase()`가 이미 전체 내부 버퍼를 공백으로 초기화하므로, 패널 영역을 별도로 `clrtoeol()`로 지우는 것은 불필요한 이중 작업이다. 제거해도 결과는 동일하고, 불필요한 ncurses 호출이 줄어든다.

#### 변경 3: (선택적 추가 최적화) `print_screen_braille`에서 빈 셀 스킵

현재 코드는 `bitmask > 0`일 때만 출력하고 빈 셀은 건너뛴다. 이것은 이미 올바르게 되어 있다. `erase()`가 배경을 공백으로 채우므로, Braille 출력이 없는 영역은 자동으로 공백이 된다. 추가 수정 불필요.

## 수정 전후 비교

### Before (`draw_screen()`)
```cpp
void Screen::draw_screen(bool no_panel) {
    clear_screen();   // logicalPixels 초기화
    project();        // 3D→2D 투영

    // ... compact_level 계산 ...

    clear();          // ★ ncurses 전체 화면 무효화
    print_screen(offset);

    for (int r = start_row; r < rows; ++r) {  // ★ 이중 지우기
        move(r, 0);
        clrtoeol();
    }
    if (!no_panel) {
        panel->draw_panel(...);
    }
    refresh();        // 전체 화면 재출력 (clear 때문에)
}
```

### After
```cpp
void Screen::draw_screen(bool no_panel) {
    clear_screen();   // logicalPixels 초기화
    project();        // 3D→2D 투영

    // ... compact_level 계산 ...

    erase();          // ✓ 내부 버퍼만 초기화, diff 최적화 유지
    print_screen(offset);

    // clrtoeol 루프 제거 — erase()가 이미 처리
    if (!no_panel) {
        panel->draw_panel(...);
    }
    refresh();        // 변경된 셀만 터미널에 출력
}
```

## 기대 효과

1. **깜빡임 제거**: `refresh()`가 변경된 셀만 출력하므로, 화면 전체가 한 번에 지워졌다가 다시 나타나는 현상이 없어진다.
2. **터미널 I/O 감소**: 변경되지 않은 영역의 데이터 전송이 생략되어, 특히 SSH 원격 환경에서 체감 성능이 크게 향상된다.
3. **패널 영역 안정**: 패널 내용이 매 프레임 지워졌다 다시 그려지는 대신, 변경된 부분만 갱신된다.

## 리스크 및 고려사항

- **잔상 문제**: `erase()` 사용 시, 이전 프레임에서 Braille 문자가 있었지만 현재 프레임에서 없는 셀은 자동으로 공백으로 덮어씌워진다 (`erase()`가 전체 버퍼를 공백으로 채우므로). 따라서 잔상은 발생하지 않는다.
- **`Panel::draw_panel()` 내부의 `clear_line()`**: 패널 내부에서 각 줄을 `move(r,left); clrtoeol()`로 지우고 다시 쓰는데, `erase()` 후에는 이미 공백이므로 문제없다. 패널 높이가 변경되어 이전보다 짧아진 경우에도 `erase()`가 전체를 공백으로 채우므로 이전 패널 잔상이 남지 않는다.
- **터미널 호환성**: `erase()`는 모든 ncurses 호환 터미널에서 지원되는 표준 함수이다.
